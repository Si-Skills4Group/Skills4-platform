Redesign the SDR module of this CRM MVP so it reflects a real SDR operating workflow based on sales-management feedback.

This is NOT a generic CRM pipeline.
This is a practical SDR workspace that is primarily phone-led, task-led, and queue-led.

CORE OPERATING MODEL

The SDR workflow should be designed around:
- call attempts
- contact outcomes
- scheduled follow-ups
- meeting booking
- qualification
- handover into the main sales / employer engagement pipeline

The module must support both:
1. marketing/email-originated leads
2. pure outbound call-led prospecting

This means the SDR module should recognise important funnel junctions such as:
- MQL (marketing qualified lead)
- Meeting Booked
- SQL (sales qualified lead)
- Opportunity Created

These junctions are critical for reporting.

PRIMARY GOAL

Build an SDR workspace where a rep can:
- work through a daily queue
- see new prospects and scheduled follow-ups in one place
- log call attempts quickly
- record whether contact was made or not
- capture call outcomes
- create follow-up tasks from real conversations
- mark leads as warm, interested, booked, qualified, bad data, do not contact, etc.
- hand qualified leads into the main engagement / opportunity pipeline

IMPORTANT PRODUCT PRINCIPLE

This module must feel like a call-led SDR cockpit, not like a traditional CRM admin page.

Optimise for:
- fast rep productivity
- low click count
- queue management
- next action visibility
- realistic SDR daily workflow

Do NOT optimise primarily for:
- record maintenance
- over-detailed forms
- marketing automation first
- admin-heavy navigation

10. FOLLOW-UP AND TASK LOGIC
--------------------------------------------------

Tasks must be central to the SDR workflow.

This is a required operating principle.

When a rep has a semi-warm call and the prospect says things like:
- call me in 4 days
- send me information
- I’m busy, follow up next week

The system must support:
- creating a follow-up task immediately
- surfacing that task in the queue
- surfacing that task on the dashboard
- linking that task to the organisation / contact / engagement

Add logic so that:
- follow-up required can be set from a call outcome
- next call date can be captured quickly
- tasks appear in “due today”, “overdue”, and “upcoming follow-ups”

11. MQL / SQL / OPPORTUNITY JUNCTIONS
--------------------------------------------------

Support key reporting junctions in the funnel.

These are especially important:
- MQL
- Meeting Booked
- SQL
- Opportunity Created

Use the following interpretation:
- MQL = marketing/email interaction has indicated possible interest
- SDR then works the lead by phone
- Meeting Booked = key conversion point
- SQL = meeting sat / confirmed as good lead
- Opportunity Created = handover into main employer engagement / opportunity flow

Support these flags and transitions clearly in the data model and UI.

  - latest outreach summary

  - follow-up context
- mark handover status
- create a follow-up task for the receiving owner

Important:
Do not create duplicate organisations or contacts.
